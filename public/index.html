02 — Как работаю

Подход к задачам

01

Понять задачу целиком

Прежде чем писать код — разобраться зачем. Часто правильный вопрос убирает половину работы.

02

Итеративно, с результатом

Рабочий прототип -> фидбек -> улучшение. Не держу незаконченные вещи в голове неделями.

03

ИИ как ускоритель

Использую OpenAI + Claude для шаблона кода, ревью кода и документации. Это не замена, а скорость.

04

Качество кода — норма

TypeScript понятные имена, маленькие функции. Код пишется один раз, а читается многократно.

⚡

ИИ в ежедневной работе

OpenAI использую для обработки большой документации, Claude — для ревью и объяснения сложных концепций. ИИ не заменяет меня в написании кода, а лишь автоматизирует вспомогательные процессы.
